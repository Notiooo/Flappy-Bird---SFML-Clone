\doxysubsection{Detailed Description}
State of bird behaviour. 

This state allows to control behaviour of a specific in-\/game object. Fully developed to break down the bird\textquotesingle{}s behaviour into individual parts. In-\/game bird has a specific flow, for example\+: 1) It starts with Flying State waiting for user input 2) As it gets input it switches to Flapping State that applies gravity and allows player to \char`\"{}flap\char`\"{} the bird with another click. 3) It can now go to Hit State -- when it hit a pipe, or go to Ground State -- when it hits ground. Both states take away the player\textquotesingle{}s ability to control the bird. The state of impact additionally forces the bird to fall down and go to Ground State. 4) Ground State displays game over, and after a while switches to \mbox{\hyperlink{class_score}{Score}} State 5) In \mbox{\hyperlink{class_score}{Score}} State it displays the score, and wait for player input so it can restart the game.

\begin{DoxyWarning}{Warning}
It is created inside \mbox{\hyperlink{class_game_state}{Game\+State}}, so actually the main \mbox{\hyperlink{class_state_stack}{State\+Stack}} created in window class still controls the whole application with this stack itself. It means if there is \mbox{\hyperlink{class_application___state}{Application\+\_\+\+State}} pushed onto the stack it has the priority. 
\end{DoxyWarning}
