\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename State\+Type$>$\newline
class State\+Stack$<$ State\+Type $>$}

A simpler version of finite state machine called State Stack. 

Made according to practices in \char`\"{}\+S\+F\+M\+L Game Development Book\char`\"{}

\mbox{\hyperlink{class_state_stack}{State\+Stack}} that stores states of the game.

In its overall operation, it allows it to act similarly to the behaviour of a stack. Certain states of the game can be overlapped or removed from the stack.

For example\+: 1) We can have \mbox{\hyperlink{class_title_state}{Title\+State}} which displays logo of the company. 2) Then \mbox{\hyperlink{class_title_state}{Title\+State}} is removed from the stack, and \mbox{\hyperlink{class_game_state}{Game\+State}} is Pushed 3) While the game works, and player is playing we can push \mbox{\hyperlink{class_pause_state}{Pause\+State}} on the stack. Now \mbox{\hyperlink{class_pause_state}{Pause\+State}} code execute first, and it can (but do not have to) block layers of stack underneath. This way we can easily make \mbox{\hyperlink{class_pause_state}{Pause\+State}} draw transparent darkish rectangle on the screen, while \mbox{\hyperlink{class_game_state}{Game\+State}} underneath is still working, or not if we want to block it inside \mbox{\hyperlink{class_pause_state}{Pause\+State}}.
